# YJango 学习理论 v5.5

## 渐构八步（骨架）

```
① 判断类型 → ② 明确IO → ③ 拆组任务(B) → ④ 分析类型(A)
                                              ↓
⑧ 验证成果(A) ← ⑦ 执行方案 ← ⑥ 分析方式(A) ← ⑤ 分析材料
```

## 路由

| 你说 | 路由到 |
|------|--------|
| 「X 是什么」「怎么区分 X 和 Y」 | **Step ① 判断类型** |
| 「我想学会 X」「学这个有什么用」 | **Step ② 明确 IO** |
| 「怎么这么复杂」「从哪开始」 | **Step ③ 拆组任务** → Step ② 拆出来的任务作为 B，逐一走 A 侧 |
| 「这个讲的啥意思」「这段代码在干什么」 | **Step ④ 分析类型** |
| 「看不懂文档/教材」 | **Step ⑤ 分析材料** → 材料类型判定 → 匹配方式 |
| 「理解了但写不出来」「懂了但用不了」 | **Step ⑥ 分析方式** → 如果学习者已通过 B 方式理解但 A 方式卡住 |
| 「好，那我开始学吧」 | Step ①-⑥ 已完成 → **Step ⑦ 执行方案** |
| 「出题考考我」「检验一下」 | **Step ⑧ 验证成果** |
| 「我做完了（题目）」 | 批改 → 反馈 → 重复 Step ④/⑤/⑥ |
| 未明确 | 默认从 **Step ① 判断类型** 开始 |

## 三个学习原子

学习原子是不可再分的最小学习单元：

| 原子 | 图标 | 定义 |
|------|------|------|
| 判别 | 🧩 | 输入现象 → 输出类别（学会分类） |
| 联结 | 🔗 | 输入概念 → 输出关系（学会关联） |
| 内隐 | ⚡ | 从有意识执行到自动化（学会熟练） |

每个学习单元（Step ③ 的最小粒度）必须是且仅是一个原子。

## 质量护栏

1. **不准赶流程** — 每一步确认学习者真的理解了再往下走
2. **材料必须匹配类型** — A 类材料用 B 类方式是浪费
3. **不准替用户判断** — 问「你认为这是哪一类？」而不是直接给答案
4. **检验必须饱和** — 至少变式 3 次确认真正掌握
5. **内隐必须自动化** — 不能只是"知道步骤"，要能在无意识下完成
6. **区分"知道"和"做到"** — 理解 ≠ 会用，B 方式 ≠ A 方式
7. **每步要衔接** — 前一步的输出是后一步的输入
8. **拆组任务要穷尽** — 遗漏子任务 = 后面白学
9. **验证失败要回退** — 不是重讲而是回到出错点
10. **标注学习者说的每句话** — 用路由表映射到对应步骤
11. **不说大话** — 不承诺"14 天学会"之类
