# 二十二、下料用法：如何用下层材料

> **核心流程**：抽象 → 判别 → 编码 → 归纳

## 核心认知过程

| 过程 | 定义 |
|------|------|
| **抽象** | 比较多个具体现象，剔除差异属性，提取共有属性 |
| **判别** | 用判别模型对具体现象进行概念归类 |
| **编码** | 用判别后的概念代替原现象（判别的应用） |
| **归纳** | 从多个输入-输出对应关系中找到共有关系 |

## 使用演示（简要）

以两个故事为例（张三讨论社会事件、韩梅申请预算）：

1. **抽象**：剔除细节差异→两个故事都可归为"主旨概述"和"细节展开"
2. **判别+编码**：将现象替换为概念→"先主旨后细节"、"先细节后主旨"等
3. **确定推测任务**：输入=表达结构，输出=理解难度
4. **归纳**：比较→"先主旨后细节→理解难度低"

## 关键特性

- 下层材料可提炼出的上层规律**不是唯一的**（材料越少，提炼方向越多）
- 抽象和归纳的结果**不必非要用语言表达**——能做到正确判别和推测即拥有模型
- 不擅长这种学习的原因：习惯了等老师划重点，面对下层材料"不知道该记什么"
