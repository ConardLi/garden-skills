# 十九、常用分析框架

## 如何分析一个学习问题

**Step 1: 分析原子构成**
- 🧩 **判别**：需要建构哪些判别模型？（"这是什么"）
  - 是常见概念，还是多义概念，还是元概念？
- 🔗 **联结**：需要建构哪些联结模型？（"有什么关系"）
  - 变量间是因果关系还是条件关系？
- ⚡ **内隐**：是否需要自动化？（"需要做到多快"）
  - 是仅限于知道，还是需要熟练到下意识完成？
- 原子间的依赖关系是什么？
  - 判别(地基) → 联结(上层) → 内隐(固化)

**Step 2: 确定下上结构**
- 对象层是什么？（要预测什么问题）
- 共象层是什么？（用什么规律来预测）
- 是经验预测还是模型预测？

**Step 3: 检查规范性**
- 目标明确吗？（要建构什么判别能力）
- 材料合格吗？（有足够的样例/清晰描述）
- 运用正确吗？（在比较共有属性/理解描述，还是在记忆材料）
- 验证有效吗？（能否处理未见情况）

**Step 4: 诊断问题**
- 学不会 → 是材料不够、方法不对、还是验证有误？
- 记不住 → 是否先建构了判别模型？
- 不会用 → 是模型没建构好，还是缺少应用经验？
- 有判无联 / 有联无判 / 判联错配？

## 知识的两个构成
```
材料的判别模型 + 概念间的推测模型(联结模型) = 知识
```
- **知识 vs 材料**：材料是食材，知识是菜肴。记下材料不等于学到知识

## 正确看课/看书方法
不是通篇记忆，而是建构两个模型：
1. 概念的判别模型（"是什么"）
2. 概念间的关系模型（"有什么关系"）

## 概念标准学习流程
1. 抽象描述
2. 多个正例（差异性越大越好）
3. 2个反例（非常像但不属于）
4. 1个类比
5. 1个验证题
