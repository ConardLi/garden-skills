# 学习材料标注指南

每次提供学习材料时，必须在材料前按以下格式标注。

## 标注模板

```
📖 学习 / 🧠 记忆 · 🧩 判别模型 / 🔗 联结模型 · 🎯 外显 / ⚡ 内隐
```

或简写为横线分隔：

```
📖 · 🧩 · 🎯
```

## 判定标准

### 学习 vs 记忆

| | 学习 📖 | 记忆 🧠 |
|--|---------|---------|
| 本质 | 构建可泛化模型 | 存储/再现已见信息 |
| 验证方式 | 能否处理未见实例 | 能否回忆起已见信息 |
| 典型场景 | 概念正反例、技能练习、策略分析 | 背定义、背参数、背代码片段 |

**注意**：同一个材料可以同时包含学习和记忆成分（比如记住某个API语法后再用概念判断），此时标注两者并用，如 `📖+🧠`。

### 判别模型 vs 联结模型

| | 判别模型 🧩 | 联结模型 🔗 |
|--|-----------|-----------|
| 功能 | 判断"是什么" | 推断"什么关系" |
| 输入 → 输出 | 现象 → 类别标签 | 概念 → 另一概念 |
| 类比 | 分类器 | 推理规则 |
| 典型材料 | 正反例对比、未见实例判别 | 关系图、因果关系、流程推理 |

### 外显 vs 内隐

| | 外显 🎯 | 内隐 ⚡ |
|--|---------|---------|
| 意识参与 | 需要刻意注意 | 自主运行 |
| 可否描述 | 可用语言表达 | 难以用语言表达 |
| 形成方式 | 讲解、分析、推理 | 大量重复、亲身体验 |
| 典型材料 | 概念讲解、规则说明 | 反复练习、肌肉记忆 |

## 常见组合示例

| 材料 | 标注 |
|------|------|
| 给出三个正例一个反例，让用户判断是否属于某概念 | 📖 · 🧩 · 🎯 |
| 解释因果关系：为什么A导致B | 📖 · 🔗 · 🎯 |
| 让用户反复练习焊接直到手顺 | 📖 · ⚡ |
| 让用户背下某个公式/参数表 | 🧠 |
| 先理解原理（概念），再用口诀记住关键点 | 📖 · 🧩 · 🎯 + 🧠 |
| 给出两种设计方案，分析适用场景差异 | 📖 · 🔗 · 🎯 |
| 让用户反复打螺丝直到不需要看就知道手感 | 📖 · ⚡ |

## emoji 角色标识规范

适当使用 emoji 美化内容呈现，提供**视觉锚点**帮助用户快速定位。

### 角色映射表

| 角色 | Emoji | 示例 |
|------|-------|------|
| 教学引导 / 解说 | 👩‍🏫 | 👩‍🏫 接下来我们看电子齿轮比怎么算 |
| 提问 / 检验 | ❓ | ❓ 如果负载突然变大，伺服会怎样？ |
| 正例 | ✅ | ✅ 17位编码器，设 8192/625 = 10000脉冲/圈 |
| 反例 | ❌ | ❌ "电子齿轮比调速度"——不对 |
| 关键结论 | 💡 | 💡 本质：它是命令脉冲到编码器位置的映射 |
| 警告/注意 | ⚠️ | ⚠️ 电子齿轮比不能无限放大，超出驱动器范围会报警 |
| 总结 | 📌 | 📌 电子齿轮比 = 编码器分辨率 ÷ 目标脉冲当量 |
| 用户操作 | 🖐️ | 🖐️ 你来试试算这个例子... |

### 使用原则

1. **一物一标，不堆砌**：每个段落/要点**最多一个开头 emoji**。禁止在一句话里塞多个 emoji。
2. **Step 标识统一**：
   - 🔴 Step ① 判断类型
   - 🟢 Step ② 明确 IO
   - 以此类推，每个 Step 用不同颜色圆点区分
   - 内容较多时用 `└─` 缩进表示子步骤
3. **结构分隔**：长时间教学用 ➕ 或 🌟 作为"里程碑标记"
4. **常见题型标记**：📖 概念讲解 / ✍️ 练习 / 🎯 检验 / 🔄 复习

### 禁止行为
- 满屏 emoji，每句话都加 → 破坏可读性
- 用 emoji 替代文字内容 → emoji 是辅助，不是主体
- 自创含义模糊的符号 → 必须使用映射表中的约定符号
