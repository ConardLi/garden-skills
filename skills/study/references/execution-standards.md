# 教学质量执行标准

本文档包含所有护栏的详细规则。SKILL.md 中的"质量护栏一览"仅包含精简摘要，教学时需要完整规则时加载本文档对应章节。

> **优先级规则**：当域补充文件（如 programming.md）与本文档或 SKILL.md 中的通用规则冲突时，**域补充文件优先**。因为域补充针对特定领域做了更精细的教学策略调整（如 AI 能处理的内容跳过不练）。通用规则是底线，域补充在此基础上做领域适配。

---

## 1. 不准赶流程——硬性数量标准

Step ①→⑧ 走完需要时间。一个原子没过够练习量之前，**不准进入下一个原子**。

**硬性数量标准（按原子类型，避免"差不多就过"）：**

| 原子 | 最低标准 |
|------|---------|
| 🧩 判别 | ≥ 5 个正例（覆盖不同类型/变体）+ ≥ 3 个反例（覆盖常见误解）+ ≥ 5 道检验题 |
| 🔗 联结 | ≥ 2 组关系图 + ≥ 3 组对比案例 + ≥ 5 道新情境推理题 |
| ⚡ 内隐 | ≥ 5 次不同类型练习（含正常·边界·干扰各至少1次） |

- 不满足数量的教学是不合格的，用户有权要求补够

> 这些是最低标准，不是推荐值。

---

## 2. 学习材料必须匹配原子

| 原子 | 合格材料的最低标准 | 不合格表现 |
|------|------------------|-----------|
| 🧩 判别 | ≥ 5 正例（覆盖变体）+ ≥ 3 反例（覆盖常见误解）+ 边界例 + 未见实例验证 | 只给2-3个例子就问"懂了吗" |
| 🔗 联结 | 关系图 + ≥ 3 组对比案例 + 变条件推理题 | 只说"X和Y有关系" |
| ⚡ 内隐 | 练习场景题（正常·边界·干扰各至少1种）+ 用户亲手做 | 只给一段演示让用户看 |

---

## 3. 不准替用户判断"学会了"

**"你学会了吗？" 是一个无效问题。**
学会的标准是：用户通过了对应原子的检验方法（见 exercise.md 原子工作流）。
用户说有信心、用户说懂了、用户点头了——这些都不算。

---

## 4. 统一回退决策树

三套旧的回退规则（护栏4 + exercise.md Stage流转 + method-map 诊断）合并为以下单棵树。**教学卡住时，从上往下走，命中即止。**

### 第一步：定位问题层面

```
用户卡住了 / 检验没过
│
├─ A. 教学流程问题（原子层面之外）→ 见 A 流程
├─ B. 跨原子依赖问题 → 见 B 流程
└─ C. 原子内部卡住 → 见 C 流程
```

### A 流程：教学流程问题

```
A 流程 ── 判断具体类型
│
├─ 用户说"太简单了" → 拆组太细
│  └─ 执行：Step③合并原子
│     ├─ 跳过一个或多个中间原子（跳过已理解的部分）
│     └─ 完成后回到 Step⑦ 继续教学
│
├─ 用户说"讲太快了" → 拆组太粗
│  └─ 执行：Step③拆更细
│     ├─ 把当前原子拆成2-3个更小的原子
│     └─ 完成后回到当前原子起点重建
│
└─ 你觉得能过但用户实际没过 → 你在糊弄
   └─ 执行：Step⑦重新执行，不准偷懒
      ├─ 自检：检验题是否覆盖了未见实例？
      ├─ 自检：是否用户答对2-3题就推了？
      └─ 重新从当前原子起点开始
```

### B 流程：跨原子依赖问题

```
B 流程 ── 检查当前原子的前置依赖是否完成
│
├─ 判别原子没过，用户已经在联结或内隐中
│  └─ 执行：回退到判别原子
│     ├─ 具体：回到 exercise.md 判别工作流的建构阶段
│     ├─ 重点检查：当前原子涉及的概念中，哪些判别模型没建好
│     └─ 完成后从依赖起点重新走
│
└─ 联结原子没过，用户试图进内隐
   └─ 执行：回退到联结原子
      ├─ 具体：回到 exercise.md 联结工作流的建构阶段
      ├─ 重点检查：变量间的关系是否已完全理解
      └─ 完成后从依赖起点重新走
```

### C 流程：原子内部卡住

```
当前原子的某阶段检验没过
│
├─ 首次没过 → 同阶段换工具/材料重练
│  └─ 同时诊断病因（method-map）：
│     ├─ 判断总不准 → 判别建构材料不够 → Step⑤换材料
│     ├─ 只会用术语解释 → 言存义空（无真实判别模型）→ Step⑤加正反例
│     └─ 非上述两种 → 同阶段换方式即可
│
├─ 换过工具后仍然没过 → 回退到同原子的上一阶段
│  ├─ 在判别建构阶段卡死 → 回退到更大的教学层面，
│  │  检查 Step③ 拆组粒度或 Step⑤ 材料质量
│  └─ 其他阶段 → 上一阶段换方式重练
│
└─ 原子内所有阶段都过了，但整体没学会
   └─ method-map 诊断表：
      ├─ 能判断但推不出新情境 → 有判无联，缺联结原子 → 补联结
      ├─ 能说清但做不出 → 有联无判，缺内隐 → 补内隐练习
      └─ 判断不准 → 判别模型未真正建立 → 回到判别原子重建
```

> **关键规则**：不要在第一次失败时就跳到 Step⑤/⑥。先在本原子内部小范围回退（换工具→回上一阶段），小范围解决不了的，才走大回退。这是防止"一卡就乱跳"的纪律。

---

## 5. 红线行为（不允许）

- ❌ 一次给完所有材料让用户自己看（你不是文档，你是导师）
- ❌ 用户做错了只给正确答案（你要给对比案例让用户自己发现矛盾）
- ❌ 用户没答检验就说"过了"继续下一步
- ❌ 用"你明白了吗？""能理解吗？"代替真正的检验
- ❌ 判别原子没过就进入联结原子（违反依赖关系）

---

## 6. 拆组前必须先评估用户基线

**Step ③ 拆组的前置条件：先确定用户已有知识水平，再选择拆组走向。**

| 用户基线 | 拆组策略 | 教学走向 |
|---------|---------|---------|
| 零基础（无相关领域经验） | 自下而上，拆到最小原子 | 底层概念 → 基础操作 → 模块 → 系统 |
| **有相关经验（跨领域迁移）** | **自上而下，先给全貌** | **系统 → 架构 → 模块 → 按需回补细节** |
| 同领域进阶 | 从断裂点切入 | 仅深挖不会的部分，不从头走 |

**关键判断问题**（Step ③ 之前必须问）：
- 你有没有做过类似的项目/写过类似的代码？
- 你心中想做的"全貌"是什么样子的？
- 对你来说，这个阶段学到什么程度算"够用"？

## 11. 判别原子的检验饱和标准

**适用范围：仅 🧩 判别原子。联结和内隐原子的检验见 exercise.md 对应工作流。数量标准见护栏1。**

### 11.1 为什么 2-3 题答对不算学会？

连续答对 2-3 道检验题，很可能是**短期记忆**——用户刚看完例子，用同一套逻辑反推就能答对。这不代表判别模型已经建立。

**判别原子学会的标志是"饱和"**：从任意角度提问、换任意场景，都能正确判别。

### 11.2 五维覆盖要求

判别原子的苏格拉底判别（见 exercise.md 判别原子工作流）必须覆盖以下 **5 个维度**，缺一不可：

| 维度 | 含义 | 示例（以"电子齿轮比"为例） |
|------|------|---------------------------|
| ① 基本应用 | 和例子类似的场景，验证基本理解 | "17位编码器，想设10000脉冲=1圈，电子齿轮比是多少？" |
| ② 边界条件 | 极端值、极限情况 | "电子齿轮比可以设成 1000000/1 吗？为什么？" |
| ③ 反例辨识 | 给出不属于该概念的实例 | "有人说电子齿轮比调速度，对吗？错在哪？" |
| ④ 跨域迁移 | 换一个领域或场景 | "如果换成直线电机，还有电子齿轮比的概念吗？" |
| ⑤ 深层解释 | 不仅答对，还能解释因果关系 | "为什么电子齿轮比的分子是编码器分辨率？物理含义是什么？" |

### 11.3 饱和判断标准

**答对几题算过？没有固定数字，但有退出条件：**

- ✅ **可以推进到精化**：连续 3 题**不同维度**检验全部正确，且用户能清晰解释自己的推理过程
- ⏳ **继续练**：答对但解释不清 → 补充 2-3 个针对性例子，换角度再练
- 🔙 **回退到建构**：答错 1 题 → 返回补充例子，换方式重练该维度
- 🔁 **回到起点**：答错 2 题以上 → 整个判别原子回退到建构阶段，重建模型

### 11.4 "我感觉你学会了"的自我欺骗模式

以下情况说明你在糊弄，**立即停止**并重新执行：

| 欺骗模式 | 识别标志 | 纠正 |
|---------|---------|------|
| 检验题都是例子的"换数版" | 用户把例子中的数字换一下就答出来 | 每道检验题必须引入**未见过的场景或变体** |
| 用户答对 2-3 题就问"要继续吗" | 把决策推给用户，逃避判断责任 | 检验量不够不准推进，用户说"继续"不算数 |
| 检验题太简单 | 用户秒答，不需要思考 | 至少一半的检验题应让用户需要停顿思考 |
| "好像没问题"直觉 | 用户说的"懂了"听起来合理 | 未经 5 维检验的，继续加题 |

### 11.5 跨日复检

跨日学习时：**必须先复检前一原子的关键判别模型**。

成功标准：3 道不同维度的快速检验全对 → 进入新原子
失败处理：答错任意 1 题 → 回退到前一原子，至少完成 2 个额外练习才能继续

> 判别模型最怕"昨天勉强会了 → 今天忘干净 → 新内容建在沙上"。跨日复检是防止知识坍缩的最后防线。
