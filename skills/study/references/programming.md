# 编程学习域补充

**仅当用户学习编程/软件开发时加载**

---

## 1. 有经验编程学习者：AI 屏障原则

对于**有编程经验的用户**学习新语言/框架：

| AI 能处理的（跳过/给代码即可） | AI 搞不定的（重点教） |
|-------------------------------|---------------------|
| 语法细节、API 参数名 | 架构设计原则（MVVM、依赖注入、SOLID） |
| 具体控件属性、布局参数 | 异步与线程安全模型 |
| 模板代码、样板代码 | 通信协议设计与状态机 |
| 简单的 CRUD 代码 | 模块拆分与项目组织 |
| 文档查询 | 调试与故障排查能力 |
| 基础 UI 布局 | 性能优化与内存管理 |

**操作方式：** AI 能干的让用户自己用 AI 解决。导师只教 AI 做不好的那层。用户说"这个 AI 能搞定"时应当认可，跳过相关练习。

## 2. 拆组建议：按 AI 脆弱度排列

对于有经验的编程学习者，优先教 AI 脆弱的部分：

```
优先（AI 做不好）：
  架构模式 → 线程模型 → 协议设计 → 调试策略

延后（AI 能辅助）：
  语法 → 具体库 API → 模板代码
```

## 3. 练习验证方式

- 基础操作：不练，用户用 AI 搞定后口头确认理解即可
- 架构/设计：用户能说清楚"为什么这么设计"就算过
- 调试/排错：用户能复现排查过程就算过
